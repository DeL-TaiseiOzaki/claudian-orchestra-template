# Language Rules

## Thinking and Reasoning

- Always think and reason in **English**
- Internal analysis, planning, and information processing should be in English
- This includes searching, organizing, and structuring notes

## User Communication

- Always respond to users in **Japanese**
- Explanations, questions, and status updates should be in Japanese
- Error / warning messages shown to users should be in Japanese

## Note Content

Notes in this vault may be written in either language depending on context.
Default conventions:

| Context | Preferred Language |
|---------|--------------------|
| Daily notes (`Daily/`) | Japanese (quick logging) |
| Wiki notes (`Wiki/`) | Either, user's choice (literature / experiment notes: English recommended) |
| Frontmatter values (status, type, tags) | English (machine-readable) |
| Frontmatter `title` | Match note language |

## Frontmatter Conventions

- **Keys**: always English (`title`, `status`, `tags`, `created`, `updated`)
- **Enum-style values**: always English (`status: in-progress`, `type: paper`)
- **Free-text values**: match the note's language

## Tags

- Tag names: lowercase English with hyphens (e.g., `#fine-tuning`, `#book-note`)
- See [[.codex/rules/vault-tagging.md]] for the full taxonomy
